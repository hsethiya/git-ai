Git AI's behavior can be configured on developer machines by writing a JSON file in the user's home directory.

On Linux and macOS, this file is located at `$HOME/.git-ai/config.json`.
On Windows, this file is located at `%USERPROFILE%\.git-ai\config.json`.

## Configuration Options

All options in `config.json` are optional and will fall back to default values if not provided.

| Option | Type | Description | Default |
| --- | --- | --- | --- |
| `git_path` | `Path` | The path to the (unaltered) `git` binary you distribute on developer machines | Whichever git is on the shell path |
| `ignore_prompts` | `boolean` | Whether prompts should be excluded from authorship logs | `false` |
| `allow_repositories` | `Path[]` | Allow `git-ai` in only these remotes | If not specified or set to an empty list, all repositories are allowed |
| `exclude_repositories` | `Path[]` | Exclude `git-ai` from these remotes | If a repository is present in both allow and exclude lists, exclusion takes precedence |

## Example Configuration

```json
{
    "git_path": "/usr/bin/git",
    "ignore_prompts": false,
    "allow_repositories": [
        "https://github.com/acunniffe/git-ai.git"
    ],
    "exclude_repositories": [
        "https://github.com/internal/private-repo.git"
    ]
}
```

## Configuration Use Cases

### Limiting to Specific Repositories

Use `allow_repositories` to enable Git AI only for approved repositories:

```json
{
    "allow_repositories": [
        "https://github.com/yourorg/product-repo.git",
        "https://github.com/yourorg/mobile-app.git"
    ]
}
```

### Excluding Sensitive Repositories

Use `exclude_repositories` to prevent Git AI from tracking specific repositories:

```json
{
    "exclude_repositories": [
        "https://github.com/yourorg/secrets-manager.git",
        "https://github.com/yourorg/compliance-docs.git"
    ]
}
```

### Privacy-First Setup

Set `ignore_prompts` to `true` to exclude all prompts from authorship logs:

```json
{
    "ignore_prompts": true
}
```
